## The way I code

I write code thinking about the person who'll maintain it (usually future me at 2 AM). That means:
- Meaningful names, not `foo` and `bar`
- Comments when the "why" isn't obvious
- Breaking things into smaller functions
- Tests for the parts that matter

Not perfect, but getting better at it.
